Explain the ways to implement preprocessing (client-side vs server-side).

Two ways you can implement preprocessing are by using LESS and SASS. On the client side, both options use variables, mixins, nesting and a slew of other operations to make resulting code leaner and more organized. With CSS preprocessors, you are able to deploy your code to a staging server and see updates instantly.

Link and comment out LESS or SASS file in the HTML file including explanation of how it was executed.

I used SASS and a program for MacOS called Codekit. I created a folder called “build” to house my scss files. I created five SASS files, four of which I imported into my style.scss code. By doing this, I was able to organize and separate elements of code like buttons and layout from each other. I used Codekit to view my changes on its active server and imported to compiled style.css file into my HTML from edits I made on the style.scss file.

Define and describes the mechanisms used in either LESS or SASS: mixins, variables, functions, operators, nested variables.

SASS uses the “$” variable to preface elements to be changed. With variables, you do not have to change a value several times. Mathematical operators along with various other symbols are used in SASS to check for different behaviors. Mixins are blocks of code that can be reused throughout the stylesheet. According to sass-lang.com, “Functions allow you to define complex operations on SassScript values that you can re-use throughout your stylesheet. They make it easy to abstract out common formulas and behaviors in a readable way.” Nesting variables in SASS is very similar to nesting a CSS stylesheet, for example, with SCSS syntax, an unordered list can be nested under navigation with less code and the “nav” repetition as it would appear in CSS syntax. Nesting also makes for a cleaner code compared to CSS.
